评测结果输出说明文档

此文档用于解释当前评测任务生成的产物（位于 outputs/task_2026030501 目录下），帮助开发者快速了解如何阅读评测报告以及排查错误。

1. 任务编号 (Task ID) 的意义与来源

目录名称 (`task_2026030501`)** 即为该次评测的唯一任务编号。
获取方式：在执行 `eval_entry.py` 脚本（或 Docker 运行）时，通过命令行参数 `--task-id` 显式指定的（例如 `--task-id task_2026030501`）。实际部署或测试时，通常由调度脚本通过“日期时间+序列号”（例如 2026年3月5日01号任务）的方式动态生成，用于将该批次的多个子测试任务（如 task_1, teleqna_gen等）汇总到同一个专属工作区归档，以便于 CI/CD 持续集成时按批次独立查阅隔离。

2. 结果目录结构

在 outputs/<task-id>/（当前为 outputs/task_2026030501/）下，会自动生成以下标准的目录结构：

outputs/task_2026030501/
├── report.md          # 综合战报，Markdown格式。汇总各子任务的准确率、耗时、状态，便于人工快速浏览整体概况。
├── report.json        # 结构化的综合评测结果。除包含整体以及 Custom/Generic 维度的评分与耗时外，还结构化存储了任务状态供程序化读取，适合作为 CI/CD 流水线的质量关卡依据。
└── details/           # 底层明细目录。集中存放当前批次中每一个任务在执行时的底层日志与错题记录。

report.json 核心字段说明
report.json 用于程序化解析，其内部包含以下重要字段：
- task_id, model, timestamp: 评测批次的基础元信息。
- avg_accuracy: 全部任务的综合平均准确率，通常作为通过/拒绝发布的基准阈值。
- summary: 包含按 custom (自定义) 与 generic (通用) 类型划分的各项统计数据，包含任务数（count）、总耗时（total_duration_sec）及分类平均准确率（avg_accuracy）。
- tasks: 一个详尽的数组，记录了每个子任务（如 task_1）的运行状态：
  - status: 是否成功（success 等）。
  - accuracy: 该任务的单项准确率。
  - duration_sec: 单项耗时。
  - returncode: 底层命令的返回值，0 表示正常。
  - details_dir: 关联的底层日志具体所在时间戳文件夹名称（如 details/20260305_133205）。
3. details 目录内容与各部分文件的意义
`details` 文件夹下包含了该批次测试中，每一个子任务（串行或并行）调度产生的独立底层执行环境。它们通常以执行时的**独立时间戳**命名对应的子目录（例如 `20260305_120359/`）。

深入任意一个时间戳目录（单次任务实例），将会存在如下结构：
details/20260305_120359/
├── configs/
│   └── 20250628_151326_29317.py 
│       # 模型任务、数据集任务和结构呈现任务对应的配置文件合成的一个统一配置
│
├── logs/
│   # 执行过程中的日志（注：命令中如果指定了 --debug，日志则会直接打印在控制台而不落盘此处）
│   ├── eval/
│   │   └── <model_name>/
│   │       └── <数据集名称>.out # 基于 predictions/ 文件夹下的推理结果，在此记录精度评测过程中的各种判分比对日志
│   └── infer/
│       └── <model_name>/ 
│           └── <数据集名称>.out # 推理过程日志（记录网络请求、API耗时、并发调度等执行链路信息）
│
├── predictions/
│   └── <model_name>/
│       └── <数据集名称>.json # 推理结果，记录推理服务返回的所有模型原始输出内容
│
├── results/
│   └── <model_name>/
│       └── <数据集名称>.json # 存放精度评测阶段经过解析与对比计算后生成的原始分数记录
│
├── summary/
│   ├── summary_<timestamp>.csv # 最终精度分数呈现（表格格式，易于二次数据分析）
│   ├── summary_<timestamp>.md  # 最终精度分数呈现（Markdown 格式，便于文档阅读）
│   └── summary_<timestamp>.txt # 最终精度分数呈现（纯文本格式）
│
└── status_tmp/
    └── tmp_<model_name>_<数据集名称>.json
        # 单条推理判分的详细状态明细 JSON 文件。
        # 包含了 origin prompt、prediction、target 以及通过 evaluator 计算完的最终对错标记，是错题分析的核心。
